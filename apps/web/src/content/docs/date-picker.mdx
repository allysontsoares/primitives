# Date Picker API Reference

A headless, unstyled date picker primitive that composes into single date, date range, and multi-select variants. Built with full keyboard and screen-reader accessibility.

## Import

```ts
import { DatePicker } from "@at5/kairo";
```

---

## Root

The main wrapper and context provider. All other components of the date picker must be descendants of the Root component.

<PropTable>
  <PropRow name="mode" type="single | range | multiple" defaultValue="single">
    Defines the selection model of the picker:
    * `single`: Allows selecting a single `Date`.
    * `range`: Facilitates selecting a start and end `Date` range.
    * `multiple`: Enables selecting multiple non-contiguous `Date` objects.
  </PropRow>
  <PropRow name="locale" type="string" defaultValue="navigator.language">
    A BCP 47 language tag (e.g. `pt-BR`, `en-US`, `ar`). Controls the order of input segments, weekday labels, month labels, separators, and calendar numbering.
  </PropRow>
  <PropRow name="value" type="Date | DateRange | Date[]">
    The controlled value of the picker. Must match the structure defined by the selection `mode`.
  </PropRow>
  <PropRow name="onValueChange" type="(value) => void">
    Event handler that triggers when the selected date or date range changes. Receives the updated value matching the current mode.
  </PropRow>
  <PropRow name="defaultValue" type="Date | DateRange | Date[]">
    The initial value of the date picker when operating in an uncontrolled state.
  </PropRow>
  <PropRow name="open" type="boolean">
    The controlled open state of the calendar view popover content.
  </PropRow>
  <PropRow name="onOpenChange" type="(open: boolean) => void">
    Callback triggered when the calendar popover content is opened or closed.
  </PropRow>
  <PropRow name="defaultOpen" type="boolean" defaultValue="false">
    The initial open state of the calendar popover content in an uncontrolled state.
  </PropRow>
  <PropRow name="minDate" type="Date">
    The minimum selectable date. Any calendar date preceding this threshold will render with `data-disabled` and block selection.
  </PropRow>
  <PropRow name="maxDate" type="Date">
    The maximum selectable date. Any calendar date following this threshold will render with `data-disabled` and block selection.
  </PropRow>
  <PropRow name="disabled" type="(date: Date) => boolean">
    A custom predicate function to disable individual calendar cells dynamically. Useful for weekends, holidays, or business logic exclusions.
  </PropRow>
  <PropRow name="readOnly" type="boolean" defaultValue="false">
    When `true`, prevents any interactive date changes while keeping inputs focusable and readable.
  </PropRow>
  <PropRow name="closeOnSelect" type="boolean" defaultValue="true (single)">
    Determines whether the popover content automatically closes once a complete selection is made. Defaults to `false` for range and multiple selection modes.
  </PropRow>
  <PropRow name="weekStartsOn" type="0 | 1 | 2 | 3 | 4 | 5 | 6" defaultValue="locale default">
    Forces the calendar grid to start on a specific day of the week. 0 represents Sunday, 1 is Monday, etc.
  </PropRow>
</PropTable>

## Input

The segmented date input wrapper that renders month, day, and year input cells in the locale-correct order. It handles keyboard arrow increments, character entries, and focus transitions.

<PropTable>
  <PropRow name="index" type="0 | 1" defaultValue="—">
    Required in `range` mode: `0` represents the start date input, and `1` represents the end date input.
  </PropRow>
  <PropRow name="segmentLabels" type="{ month?: string; day?: string; year?: string }">
    Custom accessible `aria-label` parameters for screen readers, allowing customization of individual input segments.
  </PropRow>
  <PropRow name="className" type="string">
    Applied directly to the wrapping HTML `div` group element (`role="group"`).
  </PropRow>
</PropTable>

---

## Day

A cell component representing a single calendar grid day (`role="gridcell"`). Applies dynamic data attributes based on focus, today, selection, and range limits.

### Attributes Reference

The component automatically applies the following HTML5 data-attributes for styling states:

<AttributeTable>
  <AttributeRow name="data-selected" description="Present when the specific day matches the selected date value." />
  <AttributeRow name="data-today" description="Present when the grid cell matches the current system date." />
  <AttributeRow name="data-outside-month" description="Present when the day cell resides outside the bounds of the currently viewed month." />
  <AttributeRow name="data-disabled" description="Present when the day is non-selectable due to minDate, maxDate, or custom disabled rules." />
  <AttributeRow name="data-in-range" description="Applied in range mode to day cells falling strictly between the selected start and end dates." />
  <AttributeRow name="data-range-start" description="Applied in range mode to the day cell matching the selected start date." />
  <AttributeRow name="data-range-end" description="Applied in range mode to the day cell matching the selected end date." />
</AttributeTable>

## MonthCell / YearCell

Interactive buttons rendered inside `MonthGrid` and `YearGrid` templates. They handle year-to-year or month-to-month keyboard grid navigations.

### Attributes Reference

<AttributeTable>
  <AttributeRow name="data-selected" description="Present when the month or year matches the current view state configuration." />
</AttributeTable>
